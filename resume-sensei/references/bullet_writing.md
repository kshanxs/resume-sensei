# Bullet Writing & Quantification Guidelines

Use this guide when the user wants to improve, rewrite, or add metrics to their resume bullet points. Every bullet point on a professional resume must represent an achievement rather than a duty list.

---

## 1. Action-Oriented Bullet Structure
Avoid passive phrases like "Responsible for," "Helped with," or "Worked on." Always begin with a strong, active verb and structure bullets as follows:

$$\text{Action Verb} + \text{Specific Task/Project} + \text{Method/Tool (How)} + \text{Quantifiable Result/Impact}$$

### Google's X-Y-Z Formula
Always default to this formula to draft high-impact achievements:
> **"Accomplished [X], as measured by [Y], by doing [Z]"**

*   **X (Accomplishment)**: What did you achieve or build?
*   **Y (Measurement)**: How was it measured? (Revenue, percentage increase, time saved, lines of code, volume).
*   **Z (Action/How)**: What specific actions did you take and what tools did you use?

#### Example Transformations:
*   *Before (Weak)*: "Managed social media accounts."
*   *After (Strong)*: "Grew Instagram and LinkedIn audience by 250% (5K to 17.5K followers) by designing a daily content calendar and executing 10+ influencer campaigns."
*   *Before (Weak)*: "Helped with customer service."
*   *After (Strong)*: "Improved customer satisfaction (CSAT) scores by 40% (3.2 to 4.5/5.0) by redesigning the support ticketing system and training 15 junior agents."

---

## 2. STAR / CAR Method
When drafting extensive achievements, use the STAR/CAR frameworks to extract details:

*   **Situation/Challenge**: What was the baseline or problem? (e.g., inherited underperforming platform with 95% uptime).
*   **Task**: What was the goal? (e.g., target 99.9% uptime).
*   **Action**: What did you do? (e.g., migrated 15 microservices to AWS and optimized DB indexes).
*   **Result**: What was the metric? (e.g., achieved 99.95% uptime, reducing system latency by 30%).

---

## 3. Quantification Strategies
If the user claims they "have no metrics" or "can't measure their impact," guide them to discover metrics using these five discovery categories:

1.  **Money**: Revenue generated, costs saved, budgets managed, client deal sizes.
2.  **Time**: Hours saved per week, cycles reduced, speed of code compilation, deployment frequency.
3.  **Percentages**: Uptime improvements, engagement growth, conversion lifts, error rate reductions.
4.  **Scale/Volume**: Number of concurrent projects, team sizes led, size of customer base, database records managed.
5.  **Quality**: Customer satisfaction ratings, compliance audit passes, bug resolution times.

### Estimation Techniques
When exact numbers are unavailable, instruct the user to use these estimation techniques:
*   **Conservative Bound**: Estimate slightly lower than memory to maintain credibility. (e.g., if you think you saved 100 hours, write "75+ hours").
*   **Range Estimates**: Present high and low boundaries (e.g., "Managed portfolio of 8-12 projects").
*   **Percentage of Activity**: Calculate metrics based on team proportions (e.g., "Supervised 40% of the team's ticket queue").
*   **Frequency Math**: Multiply week-level metrics out to annual totals (e.g., "Processed 30 invoices daily" becomes "Processed 7,500+ invoices annually").

## 4. Power Verbs by Category

| Category | Recommended Verbs (Use in past tense) |
| :--- | :--- |
| **Leadership** | Led, Directed, Spearheaded, Orchestrated, Supervised, Championed, Oversaw |
| **Growth** | Grew, Boosted, Scaled, Expanded, Maximized, Accelerated, Amplified |
| **Creation** | Created, Developed, Architected, Designed, Built, Launched, Pioneered |
| **Efficiency** | Streamlined, Optimized, Enhanced, Automated, Transformed, Simplified |
| **Analysis** | Analyzed, Assessed, Evaluated, Audited, Researched, Diagnosed, Forecasted |
| **Problem-Solving** | Resolved, Mitigated, Corrected, Debugged, Eliminated, Restructured |

---

## 5. Output Format for Bullet Rewrites
When improving bullet points, always structure the feedback as follows:

```markdown
## BULLET POINT IMPROVEMENTS

### Original Bullet:
"[Paste original bullet]"

### Identified Issues:
*   Uses passive phrasing ("responsible for")
*   Lacks quantifiable metrics
*   Does not show the tool or method used

### Improved Version:
"[Improved bullet using X-Y-Z formula]"

### Key Changes:
*   **Action Verb**: Changed from "Managed" to "Architected"
*   **Quantifiable Metric**: Added "60% latency reduction (from 5s to 2s)" and "500K+ DAU scale"
*   **Methodology/Tools**: Added "using React hooks and AWS Lambda microservices"
*   **Business Impact**: Demonstrates direct improvement to user experience
```
